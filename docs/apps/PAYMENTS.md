# Payments

## Зона ответственности

Обработка платежей через ЮKassa и Telegram Stars. Фиксирует факт оплаты, определяет стратегию (продлить существующий ключ или выдать новый) и уведомляет пользователя.

## Ключевые модели

- **Product** — товар для Telegram Payments API. Хранит цену в рублях и в Stars, данные провайдера (чек для ЮKassa).
- **Payment** — запись об оплате. Связывает пользователя, ключ, charge_id и провайдер.

## Сервисы

- **CreatePaymentService** — оркестратор платежа. Ищет пользователя, определяет стратегию (extend/issue), создаёт Payment, отправляет уведомление через SendNotificationService.
- **ExtendKeyService** — продлевает срок действия существующего ключа на SUBSCRIPTION_PERIOD_DAYS.

## Зависимости

Зависит от: core (декораторы, исключения), users (поиск пользователя), vds (выдача/продление ключей), notifications (уведомление об оплате).
От него зависят: бот (вызывает API после успешного платежа).
