# Notifications

## Зона ответственности

Система уведомлений и рассылок. Шаблоны сообщений хранятся в БД и управляются через Django Admin. Поддерживает персонализацию через контекстные переменные и резолверы.

## Ключевые модели

- **NotificationTemplate** — шаблон уведомления с HTML-текстом, `{переменными}`, опциональной кнопкой (URL `button_url` либо callback `button_callback_data`, напр. `my_servers`) и флагом `include_payment_buttons` для кнопки оплаты.
- **Mailing** — рассылка. Привязана к шаблону, фильтру получателей и контекстному резолверу. Отслеживает статус (DRAFT / SENDING / COMPLETED / PARTIALLY_COMPLETED / FAILED) и счётчики отправки.

## Сервисы

- **SendNotificationService** — отправляет одно уведомление по slug шаблона конкретному пользователю.
- **SendMailingService** — выполняет массовую рассылку по фильтру с обработкой ошибок и подсчётом результатов.

## Вспомогательные модули

- **selectors.py** — получение шаблонов, рассылок, фильтрация пользователей (all_active, expiring_soon, not_subscribed)
- **resolvers.py** — каркас персонального контекста для шаблонов. Сейчас активен только `ContextResolverType.NONE` (резолвер ссылки на ключ удалён вместе с reconcile-моделью — ссылки строятся ботом через «Мои серверы»).

## Зависимости

Зависит от: core (декораторы, транспорт), users (модель SystemUser), vds (селектор get_active_key).
От него зависят: vds (периодические уведомления), payments (уведомление об оплате), core (уведомление при инфра-ошибке).
