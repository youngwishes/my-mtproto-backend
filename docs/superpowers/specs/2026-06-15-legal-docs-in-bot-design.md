# Юридические документы в боте — дизайн

**Дата:** 2026-06-15
**Статус:** утверждён к реализации

## Проблема

В боте нет ссылок на пользовательское соглашение и политику конфиденциальности.
Документы опубликованы на сайте:

- Условия использования — https://mtprotokeys.ru/terms
- Политика конфиденциальности — https://mtprotokeys.ru/privacy

Нужно уведомить пользователя, что, продолжая работу с ботом, он принимает эти
правила — ненавязчиво, не ломая воронку.

## Выбранная модель: гибрид (подразумеваемое согласие)

Рассмотрены три варианта:

| Вариант | Friction | Покрытие ПД | Акцепт оферты | Код |
|---|---|---|---|---|
| Только на оплате | 0 | ❌ нет до оплаты | ✅ платящие | мин |
| Явный gate («Принимаю» + флаг) | высокий | ✅ | ✅ все | флаг + миграция + эндпоинт |
| **Гибрид (выбран)** | ~0 | ✅ с /start | ✅ платящие | только тексты + кнопки |

**Обоснование гибрида:**

- **Политика конфиденциальности ≠ оферта.** Сбор персональных данных
  (`telegram_id`, `username`, `invited_from_username`) начинается уже на `/start`,
  до оплаты. Уведомление об обработке ПД должно быть доступно в момент сбора —
  поэтому ссылка на Политику показывается с первого контакта.
- **Бесплатный период.** Часть пользователей никогда не платит (free-ключи). Если
  показывать документы только на оплате — они Условия не увидят. Футер на `/start`
  закрывает эту дыру.
- **Акцепт оферты в момент сделки.** На экране оплаты — строка «Оплачивая, вы
  принимаете…» — сильный явный акцепт оферты для платящих.

**Сознательно НЕ делаем (YAGNI):** блокирующий gate, поле `terms_accepted_at`,
миграцию для 1500+ существующих юзеров, версионирование документов, любые
изменения на бэкенде.

## Объём изменений

Только репозиторий бота (`bot/`). Тексты и одна правка клавиатуры. Бэкенд, БД,
миграции, Celery, API-эндпоинты — не затрагиваются.

### 1. Константы и футер — `bot/src/messages.py`

```python
TERMS_URL = "https://mtprotokeys.ru/terms"
PRIVACY_URL = "https://mtprotokeys.ru/privacy"

LEGAL_FOOTER = (
    f'\n<i>Продолжая пользоваться ботом, вы принимаете '
    f'<a href="{TERMS_URL}">Условия</a> и '
    f'<a href="{PRIVACY_URL}">Политику конфиденциальности</a>.</i>'
)
```

Сообщения бота рендерятся в HTML (используются теги `<b>`), поэтому инлайн-ссылки
`<a href>` отображаются как кликабельный текст — без новых кнопок.

### 2. `/start` — подразумеваемое согласие + Политика с первого контакта

В `_render_start_screen` (`bot/src/handlers/start.py`) футер приписывается к тексту
один раз, DRY для всех четырёх welcome-вариантов:

```python
text = FREE_AVAILABLE_TEXT_MAPPING.get(available_free_period) + LEGAL_FOOTER
```

Футер показывается на каждом `/start` (это одна строка курсивом, прятать не нужно).

### 3. Экран оплаты — явный акцепт оферты

В `PAYMENT_METHODS_TEXT` (`bot/src/messages.py`) добавляется строка со ссылками:

```
<i>Оплачивая, вы принимаете <a href="...">Условия</a> и
<a href="...">Политику конфиденциальности</a>.</i>
```

Без новых кнопок.

### 4. Экран «📋 Информация» — постоянный юридический хаб

В `keyboards.info()` (`bot/src/keyboards.py`):

- **Удаляется** кнопка «👀 Договор-оферта» и константа `_OFFER_URL` (ссылка на
  Google Drive PDF). Условия использования её заменяют.
- **Добавляются** две link-кнопки:

```
[ 📜 Условия использования ]        → TERMS_URL
[ 🔒 Политика конфиденциальности ]  → PRIVACY_URL
[ 🔙 Назад ]
```

URL-константы импортируются из `src.messages` (одностороннее, без циклов).

## Тестирование (TDD — тесты пишутся первыми)

`bot/tests/test_handlers.py`:

- `/start` (`cmd_start`) рендерит текст, содержащий `TERMS_URL` и `PRIVACY_URL`.
- Экран оплаты (`process_boost_paid`) рендерит текст, содержащий обе ссылки.
- `keyboards.info()` содержит кнопки с `TERMS_URL` и `PRIVACY_URL` и **не**
  содержит старый `_OFFER_URL`.

Существующие тесты `/start` и info-экрана прогоняются на отсутствие регрессий.

## Затронутые файлы

- `bot/src/messages.py` — константы URL, `LEGAL_FOOTER`, правка `PAYMENT_METHODS_TEXT`
- `bot/src/handlers/start.py` — добавление футера в `_render_start_screen`
- `bot/src/keyboards.py` — `info()`: убрать оферту, добавить две кнопки; удалить `_OFFER_URL`
- `bot/tests/test_handlers.py` — новые тесты

## Документация

требуется. При желании можно отметить наличие юридических ссылок в боте.
