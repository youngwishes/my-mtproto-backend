# Бизнес-цель

Django-бэкенд — «мозг» платформы BeatVault по продаже MTProto-прокси ключей через Telegram-бота.

## Что продаём

Пользователь получает временный MTProto-прокси-ключ. Ключ реплицируется на все активные VDS-серверы, и бот показывает ссылки (`tg://proxy?...`) для каждого из них. Подключив все серверы, пользователь получает автоматический failover: если один сервер недоступен, Telegram переключается на другой. Ссылки работают на 3 устройствах одновременно.

## Монетизация

Два способа оплаты:

| Провайдер | Цена | Механизм |
|-----------|------|----------|
| ЮKassa (Yandex) | 99 ₽/мес | Telegram Payments API |
| Telegram Stars | 80 ★/мес | Встроенная валюта Telegram |

При покупке: если у пользователя есть активный ключ — продлевается на 30 дней. Если нет — генерируется новый.

## Бесплатная ссылка

Каждый новый пользователь получает бесплатный пробный период:

| Условие | Период |
|---------|--------|
| Обычный пользователь | 30 дней |
| Пришёл по реферальной ссылке | 14 дней |
| Лимит бесплатных выдач (`FIRST_MONTH_LIMIT`) исчерпан | 7 дней |

Бесплатный период выдаётся один раз.

## Реферальная программа

Пользователь делится ссылкой `https://t.me/bot/?start=<telegram_id>`. После того как 5 приглашённых активируют свой бесплатный период, пригласивший может забрать бесплатную 14-дневную ссылку. Лимит — `REFERRAL_LINKS_LIMIT` активаций.

## Жизненный цикл ключа

1. **Создание** — бэкенд выбирает наименее нагруженный VDS, отправляет POST на FastAPI-сервис, получает секрет и домен. Токен реплицируется на все остальные активные VDS.
2. **Просмотр серверов** — экран «Мои серверы» в боте запрашивает `/api/v1/users/my-servers/`. Бэкенд генерирует `tg://proxy` ссылку для каждого активного VDS на лету (токен одинаков на всех серверах).
3. **Использование** — пользователь добавляет все серверы в Telegram. При отказе одного Telegram автоматически переключается на следующий.
4. **Перевыпуск** — кнопка «Перевыпустить ссылки» в боте. Генерируется новый секрет, обновляется на всех VDS. Кулдаун — 5 минут.
5. **Истечение** — Celery Beat ежедневно удаляет протухшие ключи с VDS, деактивирует записи в БД, уведомляет пользователей.

## Уведомления

| Время (UTC) | Что происходит |
|-------------|---------------|
| 15:00 | Предупреждение за 1 день до истечения |
| 08:00 | Предупреждение за 1 час до истечения |
| 09:00 | Удаление истёкших ключей с VDS |
