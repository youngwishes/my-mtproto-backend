# Бизнес-цель

Django-бэкенд — «мозг» платформы BeatVault по продаже MTProto-прокси ключей через Telegram-бота.

## Что продаём

Пользователь получает временный MTProto-прокси-ключ — единый секрет, валидный на всём флоте серверов. Секрет асинхронно доставляется на все здоровые VDS-серверы, и бот показывает ссылки (`tg://proxy?...`) для каждого из них. Подключив все серверы, пользователь получает автоматический failover: если один сервер недоступен, Telegram переключается на другой. Ссылки работают на 3 устройствах одновременно.

## Монетизация

Два способа оплаты:

| Провайдер | Цена | Механизм |
|-----------|------|----------|
| ЮKassa (Yandex) | 99 ₽/мес | Telegram Payments API |
| Telegram Stars | 80 ★/мес | Встроенная валюта Telegram |

При покупке: если у пользователя есть активный ключ — продлевается на 30 дней. Если нет — генерируется новый.

## Бесплатная ссылка

Каждый новый пользователь получает бесплатный пробный период:

| Условие | Период |
|---------|--------|
| Обычный пользователь | 30 дней |
| Пришёл по реферальной ссылке | 14 дней |
| Лимит бесплатных выдач (`FIRST_MONTH_LIMIT`) исчерпан | 7 дней |

Бесплатный период выдаётся один раз.

## Реферальная программа

Пользователь делится ссылкой `https://t.me/bot/?start=<telegram_id>`. После того как 5 приглашённых активируют свой бесплатный период, пригласивший может забрать бесплатную 14-дневную ссылку. Лимит — `REFERRAL_LINKS_LIMIT` активаций.

## Жизненный цикл ключа

1. **Создание** — чистая запись в БД: бэкенд генерирует случайный секрет и создаёт строку `MTPRotoKey` (без выбора сервера). Затем Celery-таск `push_key_to_servers_task` идемпотентно доставляет секрет на все здоровые VDS. Глобальный потолок активных ключей — `GLOBAL_KEYS_LIMIT` (при превышении — `KeysLimitReached`).
2. **Просмотр серверов** — экран «Мои серверы» в боте запрашивает `/api/v1/users/my-servers/`. Бэкенд генерирует `tg://proxy` ссылку для каждого активного VDS на лету (секрет одинаков на всех серверах).
3. **Использование** — пользователь добавляет все серверы в Telegram. При отказе одного Telegram автоматически переключается на следующий.
4. **Перевыпуск** — кнопка «Перевыпустить ссылки» в боте. Генерируется новый секрет, запись обновляется в БД, новый секрет асинхронно доставляется на все здоровые VDS. Кулдаун — 5 минут.
5. **Истечение** — Celery Beat ежедневно удаляет протухшие ключи с VDS, деактивирует записи в БД, уведомляет пользователей.

> БД — единственный источник правды; присутствие секретов на серверах — производный кэш. Любое расхождение устраняется reconcile-механизмами: мгновенный пуш при выдаче/перевыпуске и бэкфилл (`sync_keys_to_vds_task`) при восстановлении упавшего сервера.

## Уведомления

| Время (UTC) | Что происходит |
|-------------|---------------|
| 15:00 | Предупреждение за 1 день до истечения |
| 08:00 | Предупреждение за 1 час до истечения |
| 09:00 | Удаление истёкших ключей с VDS |
